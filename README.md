# 智慧双碳管理系统 - 后端服务
## 项目简介
智慧双碳管理系统后端服务是面向**政府、园区、企业**的双碳（碳达峰、碳中和）数字化管理平台核心支撑系统，基于**Spring Boot 模块化单体架构**设计，实现碳排放数据采集、核算、监测、分析、减排管控、碳资产交易、政策适配等全场景能力。

系统遵循国家碳核算标准（GHG Protocol、ISO14064、各行业核算指南），对接IoT设备、能源系统、业务系统，为碳管理提供精准数据支撑、智能决策分析和全流程数字化管控。

## 核心功能
### 1. 基础管理模块
- 组织架构管理（政府/园区/企业多级权限）
- 用户管理、角色权限、菜单配置
- 字典管理、系统配置、日志管理

### 2. 碳数据采集与管理
- 多维度数据采集：能源消耗、工业生产、交通、建筑、废弃物等
- 支持手动录入、Excel导入、IoT设备自动采集、第三方系统接口对接
- 数据校验、清洗、溯源、归档

### 3. 碳核算与计量
- 覆盖范围一/二/三碳排放精准核算
- 内置国家/行业碳排放因子库，支持自定义因子配置
- 按日/月/年/区域/产品线维度自动生成碳核算报告

### 4. 碳监测与预警
- 实时碳排放数据可视化监测
- 异常排放、超标排放智能预警
- 能耗-碳排放关联分析

### 5. 碳减排与目标管理
- 碳达峰、碳中和目标制定与分解
- 减排项目管理、减排量核算、减排进度跟踪
- 节能改造、绿电替代、碳汇项目管理

### 6. 碳资产与交易管理
- 碳配额管理、CCER（国家核证自愿减排量）管理
- 碳交易行情对接、交易记录、碳资产估值
- 碳成本分析、减排经济效益测算

### 7. 数据分析与决策支持
- 多维度碳数据可视化大屏（园区/企业/区域）
- 碳排放趋势分析、同比/环比分析、结构分析
- 碳减排模拟、情景预测分析

### 8. 接口服务
- 第三方设备/系统对接接口
- 政府监管平台上报接口
- 前端API、开放平台接口

## 技术栈
### 核心框架
- 基础框架：Spring Boot 2.7.x / 3.x
- ORM框架：MyBatis-Plus
- 权限框架：Spring Security + JWT
- 接口文档：Swagger / Swagger-UI（原生）
- 定时任务：Quartz

### 数据存储
- 关系型数据库：MySQL 8.0+
- 缓存中间件：Redis
- 时序数据库：InfluxDB/TDengine（碳数据/物联网数据存储）

### 工具&环境
- JDK 1.8+ / 11+
- Maven 3.6+
- Git

## 项目结构
```
carbon-management-system/
├── carbon-common         # 公共模块（工具类、常量、通用配置、异常处理）
├── carbon-framework      # 框架核心模块（统一返回、拦截器、基础配置）
├── carbon-generator      # 代码生成模块
├── carbon-quartz         # 定时任务模块（数据采集、核算任务调度）
├── carbon-system         # 系统管理模块（用户、权限、组织、字典）
├── carbon-business       # 双碳核心业务模块
│   ├── data-collect      # 碳数据采集
│   ├── calculate         # 碳核算计量
│   ├── monitor           # 碳监测预警
│   ├── reduce            # 碳减排管理
│   ├── asset             # 碳资产交易
│   └── analysis          # 数据分析
├── carbon-admin          # 项目启动入口（主服务）
└── doc                   # 项目文档（SQL、接口说明、部署指南）
```

## 环境搭建
### 1. 环境准备
1. 安装 JDK 1.8+、Maven 3.6+、MySQL 8.0、Redis
2. 配置本地环境变量，确保 Java、Maven 命令可用

### 2. 数据库初始化
1. 创建数据库 `carbon_system`
2. 执行 `doc/sql/carbon_init.sql` 初始化表结构和基础数据

### 3. 配置修改
1. 修改 `carbon-admin/src/main/resources/application.yml`
2. 配置数据库连接、Redis、服务端口、Swagger 开关等信息

### 4. 项目启动
```bash
# 1. 克隆项目
git clone [项目仓库地址]

# 2. 安装依赖
mvn clean install

# 3. 启动项目
运行 carbon-admin 模块下的 CarbonAdminApplication 主类
```

### 5. 访问地址
- 项目端口：默认 8080
- Swagger 接口文档：http://localhost:8080/swagger-ui.html
- 登录接口：获取 Token 后调试接口

## 部署方式
### 1. 原生打包部署
```bash
# 1. 打包（跳过测试）
mvn clean package -Dmaven.test.skip=true

# 2. 运行 jar 包
java -jar carbon-admin.jar
```

### 2. 服务器部署
1. 将打包后的 `carbon-admin.jar` 上传至服务器
2. 配置服务器 JDK 环境
3. 使用 `nohup` 后台运行：
```bash
nohup java -jar carbon-admin.jar > catalina.out 2>&1 &
```

## 开发规范
1. 代码遵循 Alibaba Java Coding Guidelines
2. 接口统一返回标准格式：code、msg、data
3. 全局异常统一处理，自定义业务异常
4. 数据库字段使用下划线命名，代码使用驼峰命名
5. 核心业务逻辑、复杂方法必须添加注释
6. 定时任务、接口、工具类统一归类存放
